01 / Executive summary

What changed and why it matters

Reviewer attention needed
Good direction

Change thesis

The PR moves Bridge virtual account provisioning from a Tron/hardcoded-address path toward an IBEX-managed ETH-USDT Cash Wallet. Receive addresses should be wallet-bound, not fallback constants.

Main concern

Reviewer focus

The biggest review question is migration safety for accounts that already have bridgeEthereumAddress or existing virtual accounts. The new USDT wallet is provisioned before the idempotency guard, but existing address reuse can bypass new receive-info creation.

05 / Risk review

Where this can break trust

Funds routing

Existing address ownership

Existing bridgeEthereumAddress may not be bound to the newly ensured USDT wallet. Validate or migrate explicitly.

Migration

Default wallet mutation

The PR can change default wallet during VA creation even when a VA already exists.

Financial correctness

Currency-unit conversion

getAccountDetails converts based on caller currency, not IBEX response metadata.

Webhook compatibility

Tron → Ethereum cutover

Confirm no active deposits still arrive on Tron rails.

Reliability

Withdrawal concurrency

Simultaneous withdrawal requests need DB-level uniqueness or transactional protection.

Security posture

Hardcoded address removed

Strong improvement.
